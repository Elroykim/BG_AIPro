# BG_AIPro 블로그 자동화 시스템 아키텍처

## 시스템 개요

멀티 주제 · 멀티 플랫폼 블로그 콘텐츠를 AI로 생성하고, 최소 개입으로 발행·관리하는 자동화 파이프라인.

## 파이프라인

```
┌─────────────────────────────────────────────────────────┐
│                    1. PLAN (기획)                         │
│  주제 선정 → 키워드 리서치 → 콘텐츠 캘린더 생성            │
└──────────────────────┬──────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────┐
│                   2. CREATE (생성)                        │
│  아웃라인 → 본문 작성 → SEO 최적화 → 이미지 생성           │
└──────────────────────┬──────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────┐
│                  3. REVIEW (검토)                         │
│  품질 검증 → [사람 승인 대기 or 자동 승인] → 수정 반영      │
└──────────────────────┬──────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────┐
│                 4. PUBLISH (발행)                         │
│  WordPress 발행 ─┐                                       │
│  Hugo 빌드/배포 ──┤                                       │
│  네이버 크로스포스팅 ┤                                     │
│  티스토리 크로스포스팅┘                                     │
└──────────────────────┬──────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────┐
│                 5. MONITOR (모니터링)                      │
│  트래픽 추적 → 성과 분석 → 피드백 → 다음 기획에 반영        │
└─────────────────────────────────────────────────────────┘
```

## 멀티 주제 운영 구조

각 주제(토픽)는 독립된 설정을 가지며, 공통 파이프라인을 공유한다.

```
topics/
├── tech/           # 기술 블로그
│   └── config.yaml
├── finance/        # 재테크/금융
│   └── config.yaml
└── lifestyle/      # 라이프스타일
    └── config.yaml
```

topic config 예시:
```yaml
name: tech
display_name: "테크 블로그"
platforms:
  - wordpress
  - hugo
  - naver
keywords_strategy: long_tail
posting_frequency: "3/week"
tone: professional
language: ko
```

## 멀티 플랫폼 발행 전략

| 플랫폼 | 역할 | 발행 방식 |
|--------|------|----------|
| WordPress | 메인 허브 #1 | REST API 직접 발행 |
| Hugo + GitHub Pages | 메인 허브 #2 | Markdown → git push → CI/CD 자동 빌드 |
| 네이버 블로그 | 한국 SEO 채널 | XML-RPC 크로스포스팅 (생성 전용) |
| 티스토리 | 한국 보조 채널 | Selenium/Bridge API 크로스포스팅 |

## AI 엔진 구조

```
ai/
├── providers/
│   ├── claude.py        # Claude API (Anthropic)
│   ├── local_llm.py     # 로컬 LLM (Ollama 등) - 향후
│   └── base.py          # 공통 인터페이스
├── prompts/
│   ├── outline.py       # 아웃라인 생성 프롬프트
│   ├── writer.py        # 본문 작성 프롬프트
│   ├── seo.py           # SEO 최적화 프롬프트
│   └── review.py        # 품질 검토 프롬프트
└── images/
    └── generator.py     # 이미지 생성 (DALL-E, SD 등)
```

## 사람 개입 포인트

자동화 기본이되, 다음 지점에서 선택적 개입 가능:
- **콘텐츠 캘린더 확인**: 주간 계획 리뷰
- **발행 전 검토**: 승인 모드 (auto / manual / hybrid)
- **성과 리뷰**: 주간/월간 리포트 확인 후 전략 조정

## 기술 스택

- **언어**: Python 3.11+
- **패키지 관리**: Poetry
- **스케줄링**: APScheduler (로컬) / GitHub Actions (CI/CD)
- **데이터 저장**: SQLite (콘텐츠 메타데이터, 발행 이력)
- **설정 관리**: YAML config + .env (시크릿)
- **AI**: Claude API → 향후 로컬 LLM 병행
- **Hugo**: CLI 연동으로 정적 사이트 빌드
- **WordPress**: REST API 클라이언트
